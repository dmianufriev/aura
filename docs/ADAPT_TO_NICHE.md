# Адаптация AURA под другие ниши

> AURA — это **не «салонная CRM»**. Это операционка для любого **сервисного бизнеса с записью к специалистам + дополнительными продажами + удержанием клиентов**.
>
> Подходит для: стоматологии, мед.центра, фитнеса, барбершопа, ногтевой студии, автосервиса, репетитора/школы, ветклиники, тату-салона, психолога, массажиста, спа, химчистки, ремонтной мастерской, юриста, бухгалтерского обслуживания и т.п.
>
> Этот документ — **готовые конфиги под 6 ниш**. Скопировал → вставил → запустил `npm run dev` → готово к показу клиенту.

---

## Универсальная модель

Любая ниша мапится на сущности AURA так:

| AURA-сущность | Что это в реальности |
|----------------|----------------------|
| **Master** | Специалист — врач, тренер, барбер, мастер, преподаватель, механик |
| **Service** | Услуга — приём, тренировка, стрижка, ремонт, урок |
| **Client** | Клиент — пациент, ученик, гость |
| **Appointment** | Запись — приём, слот |
| **Inventory** | Расходники — материалы, медикаменты, запчасти |
| **Loyalty** | Удержание — бонусы, абонементы, депозиты |
| **AI-Inbox** | AI отвечает в WhatsApp/Direct/Telegram |
| **Reactivation** | Возврат «потерявшихся» |
| **Marketing** | Триггерные кампании |
| **Reviews** | Отзывы с AI-фильтром негатива |
| **Schedule** | Журнал записи |
| **Finance** | P&L + ЗП специалистов |
| **Analytics** | Cohort + LTV + ABC |

**Что меняется при адаптации:**
1. Названия и палитра в `salon.config.ts`
2. Имена и специализации мастеров
3. Услуги и их цены (порядок цен!)
4. Тексты диалогов в `conversations.json` и `client_scenarios.json`
5. Расходники в `inventory.json`
6. Категории в Settings → Услуги

**Что НЕ меняется (универсальная инфраструктура):**
- Архитектура, роутер, контекст роли
- UI-примитивы
- Логика стейт-машины чата
- Структура страниц
- AI-фильтры, A/B-тесты, cohort retention

### Особенности интерактива

**Что важно показать стоматологии (поменять акценты в UI):**
- **Электронная карта пациента** — расширить Sheet клиента: добавить таб «Анамнез» с заметками и противопоказаниями
- **Фото до/после** — заместо `visitHistory` имеет смысл показывать снимки
- **Депозиты и рассрочка** — стоматология часто продаёт планы лечения 50-300к, депозит критически важен
- **AI-инбокс** — отдельный сценарий «Спрашивают цену импланта»: бот даёт диапазон + предлагает консультацию
- **Программа лояльности** — бонусы за рекомендацию (реферальная) дают 30% новых пациентов

### Что менее релевантно
- Cohort retention с месячной разбивкой — у стоматологии цикл возврата 6-12 мес, лучше показать поквартально
- Загрузка слотов <50% это норма для хирурга (длинные процедуры) — не подсвечивать красным
- «Спящие 30 дней» — заменить на «60/90/180/365» (нормальный цикл — раз в полгода)

## Универсальные принципы при адаптации

### 1. Адаптируй порядок цифр
- Средний чек **барбершоп** 2.4к → **салон** 6к → **стоматология** 18к → **автосервис** 12к → **репетитор** 2.4к
- Соответственно поправь `revenue`, `appointmentsCount`, чтобы выручка соответствовала.

### 2. Адаптируй cohort retention
- **Высокочастотные** (барбершоп, маникюр, фитнес): нормально 60-80% на M+1, 40% на M+3
- **Среднечастотные** (салон красоты, репетитор): 50% / 30%
- **Низкочастотные** (стоматология, авто): 20% на M+1, но 65% на M+6

### 3. Адаптируй reactivation окна
| Ниша | Окно «потерян» |
|------|----------------|
| Барбершоп | 30/45/60 дней |
| Маникюр | 30/45/60 дней |
| Салон красоты | 30/60/90 дней |
| Фитнес | 14/30/60 дней (с пропуска тренировки) |
| Стоматология | 180/365/730 дней |
| Автосервис | 180/365 дней |
| Репетитор | 14/30/60 дней пропуска |

### 4. Адаптируй каналы коммуникации
- **B2C услуги (салон, маникюр, фитнес):** WhatsApp + Instagram = 80% трафика
- **Медицина:** WhatsApp + телефон (старшая аудитория)
- **Авто:** WhatsApp + телефон
- **B2B (репетитор корпоративный):** Email + Telegram
- **Школа для родителей:** WhatsApp + школьный чат

### 5. Адаптируй интеграции в Settings
| Ниша | Особые интеграции вместо YClients |
|------|-----------------------------------|
| Стоматология | Инфоклиника, IDENT, БИТРИКС24, ОМС-портал |
| Фитнес | МойФитнес, Fit-Service, СБИС |
| Медцентр | МИС Медиалог, qMS, Renovatio |
| Автосервис | АвтоДилер, AutoIntellect, Кросс |

## Чеклист адаптации (полный)

При смене ниши проходим по списку:

- [ ] `salon.config.ts` — название, эмодзи, телефон, адрес, средний чек, ЗП админа, палитра
- [ ] `index.html` → `<title>`, `<meta description>`, og-теги
- [ ] `masters.json` — 4 специалиста с правильной специализацией, цифрами, маржой
- [ ] `services.json` — каталог услуг с правильным порядком цен
- [ ] `dashboard.json` — KPI, источники, точки графика (можно регенерировать через `gen-mocks.mjs` с новыми мастерами)
- [ ] `insights.json` — 3 инсайта **специфичных для ниши** (не «поднять цену окрашивания», а «назначить осмотр после имплантации»)
- [ ] `clients.json` — пересоздать через `gen-mocks-extra.mjs` с новой логикой сегментации (поправь окна sleeping)
- [ ] `appointments.json` — записи к новым мастерам
- [ ] `conversations.json` — 12 диалогов с типичными для ниши вопросами клиентов
- [ ] `client_scenarios.json` — 4 сценария чата под ниши (запись на МРТ vs стрижку — разный flow)
- [ ] `lost_clients.json` — пересчитать сегменты под цикл возврата ниши
- [ ] `loyalty.json` — программы (для фитнеса абонементы важнее бонусов; для стоматологии — депозит)
- [ ] `campaigns.json` — кампании специфичные для ниши
- [ ] `inventory.json` — расходники ниши (медикаменты / запчасти / спортпит — что есть)
- [ ] `payroll.json` — пересчитать ставки (у врачей ставка ниже, у барберов 60%, у репетиторов 50-70%)
- [ ] `finance.json` — пересчитать P&L (другие категории расходов: аренда кабинета врача vs зала фитнеса)
- [ ] `cohorts.json` — retention характерный для ниши
- [ ] `reviews.json` — отзывы с правильными каналами и контекстом
- [ ] `integrations.json` — добавить специфичные интеграции, удалить нерелевантные

Большая часть моков **регенерируется одним скриптом** — если правильно поправить `scripts/gen-mocks-extra.mjs`, можно за 20 минут получить готовый набор.

## Минимальная кастомизация (чтобы показать «вашему клиенту»)

Если нет времени/желания глубоко перерабатывать моки — **минимальный список:**

1. `salon.config.ts` → название/эмодзи/палитра клиента (5 мин)
2. `masters.json` → 4 имени с фотографиями (10 мин)
3. `dashboard.json` → подставить **реальную** выручку из CRM клиента, его источники (15 мин)
4. `insights.json` → 1-2 инсайта на основе анализа его данных (30 мин — самое ценное!)
5. Билд + деплой (5 мин)

**Итого: 1 час → готова персональная демка для встречи с клиентом.** Остальные 12 разделов показывают «общий функционал», в инсайтах и дашборде — **его** цифры. Это закрывает 80% эффекта продажи.
