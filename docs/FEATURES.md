## 1. Дашборд (`/`)

**Назначение:** за 5 секунд показать владельцу, что у него происходит.

**Секции (сверху вниз):**
1. **Приветствие + позиционирование** — «AI заменил администратора 60к/мес»
2. **4 KPI-карточки**: Выручка месяца, Записей, Средний чек, Новых клиентов. С дельтой в % к прошлому месяцу (зелёная/красная стрелка).
3. **График выручки за 30 дней** — Recharts AreaChart с градиентом `--color-primary`.
4. **Рейтинг мастеров** — таблица с маржой как прогресс-бар. Маржа <30% → красный фон строки (мастер «Катя Романова» с маржой 12%).
5. **3 AI-инсайта** — карточки с суммой в ₽:
   - «Поднять цену окрашивания на 500 ₽ → +18 000 ₽/мес»
   - «Перевести 12 ч/нед с Кати на Алину → +120 000 ₽/мес»
   - «Запустить рассылку 47 пропавшим → +210 000 ₽/мес» (кнопка `Перейти к рассылке` → `/reactivation`)

**Моки:** `dashboard.json` (KPI + deltas + revenueByDay + sources), `masters.json` (рейтинг), `insights.json` (3 инсайта).

**Связи:**
- Инсайт #3 → `/reactivation` через `actionLink`
- Цифры выручки рассчитываются из `masters.json` (в скрипте генерации)

## 2. Журнал записи (`/schedule`)

**Назначение:** календарь дня владельца — кто к кому записан, какие слоты свободны.

**UI:**
- Шапка: переключатель дня (вчера/сегодня/завтра + chevrons), статистика (записей / выручка / загрузка %)
- Кнопка `Новая запись` → диалог
- Grid таблица: первая колонка — часы (10:00–20:00), остальные колонки — мастера с аватарами и специализациями
- В ячейках — карточки записи цвета по статусу:
  - `confirmed` — primary с левым бордером
  - `pending` — amber
  - `completed` — emerald
  - `no_show` — красный + line-through
  - `cancelled` — серый
- Высота карточки = `Math.ceil(duration / 60) * 56px`
- Свободные слоты — пустые ячейки, hover показывает `+`, клик → диалог создания

**Клик по записи → Sheet справа:**
- Аватар клиента + кнопки `Позвонить` / `WhatsApp`
- Детали (услуга, мастер, время, цена, предоплата)
- Заметка-аллергия в amber-блоке (если есть)
- AI-помощник «Отправлю напоминание за 24 ч и 2 ч»
- Действия: `Подтвердить` / `Перенести` / `Отменить` — все с toast

**Клик `Новая запись` → Dialog:**
- Поля: Клиент / Услуга / Мастер / Дата / Время
- Блок «💡 AI рекомендует» — подсказка по свободным слотам
- `Создать запись` → toast «Запись создана · Клиенту отправлено подтверждение в WhatsApp»

**Моки:** `appointments.json` (20 записей на сегодня), `masters.json`.

**Связи:**
- Каждая `Appointment.clientId` ссылается на клиента из `clients.json`
- `Appointment.masterId` → `masters.json`

## 3. Клиенты (`/clients`)

**Назначение:** база клиентов с сегментацией, LTV, историей визитов.

**UI:**
- 5 stat-карточек по сегментам: VIP / Постоянные / Новые / Спящие / Потеряны
- Search-инпут + фильтр-табы по сегментам
- Таблица: Клиент (с аватаром и иконкой ⚠ если в чёрном списке) / Сегмент (Badge) / Визитов / Средний чек / LTV / Бонусы / Любимый мастер / Последний визит
- Строка чёрный список — `bg-red-50`
- Показывается 30 первых, остальное через поиск

**Клик строки → Sheet:**
- Большой аватар + день рождения + первый визит + любимый мастер
- 4 stat-карточки: Визитов, Средний чек, LTV (зелёным), Бонусы + депозит
- Жёлтая заметка администратора (если есть): «Аллергия на парабены — серия Davines OI»
- Предпочитаемые услуги (Badge'и)
- **История визитов** — список последних 8 с датой/услугой/мастером/суммой/⭐ оценкой
- Кнопки: `Записать` / `Написать` / `Позвонить` / `Начислить бонусы` — toast'ы

**Моки:** `clients.json` (60 шт).

**Сегментация (в скрипте генерации):**
```js
// i — индекс клиента
if (i % 13 === 0) return "vip";
if (i % 7 === 0) return "lost";
if (i % 5 === 0) return "sleeping";
if (i % 4 === 0) return "new";
return "regular";
```

**Связи:**
- LTV рассчитан как `totalVisits × averageCheck + бонус для VIP`
- Связан с `Appointments` (visitHistory)
- Связан с `Reviews` (clientName matched)
- Связан с `Marketing` (audience)

## 4. AI-Инбокс (`/inbox`)

**Назначение:** показать, что AI отвечает в Direct / WhatsApp / Telegram, освобождая администратора.

**UI:**
- Заголовок + позиционирование
- Card-баннер метрик: «103 диалога · 67 записей · 0 пропущенных»
- Табы-фильтр: Все / Записались / В процессе / Нужен человек
- Лента карточек диалогов:
  - Аватар клиента, имя + иконка канала (WhatsApp/Instagram-camera/Telegram-send)
  - Последнее сообщение клиента (1 строка)
  - `🤖 AI: ...` — ответ AI
  - Время + Badge статуса

**Клик карточки → Sheet справа:**
- Шапка с именем, каналом, статусом
- Чат-вид: сообщения клиента (серый, слева) + AI (primary opacity 0.1, справа, с лейблом «AI»)
- Скролл при большой переписке

**Моки:** `conversations.json` (12 диалогов с разными сценариями — запись/FAQ/перенос/отмена/реактивация).

**Связи:**
- Реалистично представляет, как AI бот может отвечать клиентам из `clients.json`

## 5. Возврат клиентов (`/reactivation`)

**Назначение:** реактивация спящих клиентов с конкретными суммами потерь.

**UI:**
- Шапка-CTA: «47 потерянных клиентов · Потенциал: 282 000 ₽» + кнопка `Запустить волну рассылок`
- Табы-фильтр по группам: Все 47 / 30+ дней (12) / 60+ дней (19) / 90+ дней (16)
- Таблица: чекбокс / Клиент / Последняя услуга / Дней назад / Средний чек / Рекомендуемое сообщение (truncate + tooltip полного текста)

**Клик `Запустить волну рассылок` → Dialog:**
- 3 вкладки по сегментам с превью текста + список первых 8 имён:
  - 30+: «{имя}, давно вас не было ✨ Записать к Алине?»
  - 60+: «...прошло 2 месяца, скидка 15%...»
  - 90+: «...скучаем 💔 скидка 20%...»
- `Отправить (демо)` → закрывает Dialog + toast «47 сообщений отправлено · Ожидаемый возврат: 12 клиентов · +71 880 ₽»

**Расчёт ожидаемого возврата (хардкод):**
- 30+: 35% × 12 × 5990 ≈ 25к
- 60+: 25% × 19 × 5990 ≈ 28к
- 90+: 15% × 16 × 5990 ≈ 14к
- Итого: 12 клиентов · 71 880 ₽ (в коде значение зашито)

**Моки:** `lost_clients.json` (47).

**Связи:**
- AI-инсайт #3 на Dashboard ведёт сюда
- В будущем — Marketing→sleeping_30/60/90 кампании могут отправлять то же самое

## 6. Маркетинг (`/marketing`)

**Назначение:** триггерные авто-кампании, статистика по конверсии и A/B-тесты.

**UI:**
- 3 stat-карточки: Отправлено за месяц / Средняя конверсия / Принесли выручки (с дельтой)
- Список 7 кампаний:
  - Иконка `Megaphone` + название + Badge `A/B` если включён A/B-тест
  - Канал (иконка + название) + триггер
  - 4 метрики: Аудитория / Open rate / Конверсия / Выручка
  - Switch вкл/выкл (toast)
  - Кнопка `Открыть` → Dialog

**Шаблонные кампании в моках:**
1. Welcome series (first_visit, WhatsApp) — 41% конверсия
2. День рождения (birthday) — 75% конверсия
3. После визита — собрать отзыв (after_visit, Telegram)
4. Спящие 30+ дней (с A/B-тестом: без скидки vs −10%, B победил)
5. Спящие 60+ дней
6. Спящие 90+ дней
7. Годовщина первого визита (выключена)

**Dialog кампании:**
- Превью текста с переменными {имя}, {мастер}, {услуга}
- Блок A/B-теста (если есть): два варианта, победитель — emerald фон
- 3 stat-карточки
- Кнопки: `Закрыть` / `Редактировать` / `Тестовая отправка` → toast

**Моки:** `campaigns.json`.

**Связи:**
- Аудитории пересекаются с `lost_clients.json` (sleeping 30/60/90)
- Конверсия → закрывает Reactivation как процесс

## 7. Отзывы (`/reviews`)

**Назначение:** AI-модерация отзывов. Негатив → админу, позитив → автопубликация.

**UI:**
- 4 stat-карточки: Средняя оценка (со звёздами) / Всего / Положительных (%) / Требуют ответа (красный)
- Card с переключателем «Автопубликация ≥4★»
- Табы: Все / Негатив (≤3★) / Нужна реакция
- Список карточек отзывов:
  - Аватар клиента + канал + услуга + мастер + дата
  - 5 звёзд (заполненные/пустые)
  - Badge статуса
  - Текст отзыва
  - Если есть AI-предложение ответа — primary блок «🤖 AI предлагает ответ» с цитатой
  - Кнопки: `Ответить` / `Опубликовать` (для ≥4★) / `Передать админу` (для ≤2★)

**Клик `Ответить` → Dialog:**
- Цитата отзыва со звёздами
- Textarea с AI-шаблоном ответа (по умолчанию)
- Для негатива — amber-блок «Рекомендация AI: предложить компенсацию в личной беседе»
- `Опубликовать ответ` → toast «Ответ опубликован на Яндекс.Карты»

**Моки:** `reviews.json` (10: 6 положительных, 1 нейтральный, 3 негативных с шаблонами ответов).

**Связи:**
- Отзывы привязаны к клиентам из `clients.json` (clientName)

## 8. Лояльность (`/loyalty`)

**Назначение:** 4 типа программ удержания: бонусы, депозит, сертификаты, абонементы.

**UI:**
- Шапка + `Создать программу`
- 4 карточки программ с иконками (Gift/Wallet/Award/RefreshCw):
  - Название + правило + описание
  - Switch вкл/выкл
  - 3 метрики: Участников / Выдано (₽) / Использовано (₽, зелёным)
- Внизу — табы:

**Таб «Сертификаты»** — таблица 15 сертификатов:
- Код (моноширинный AURA-XXXX)
- Номинал (3к/5к/10к/15к)
- Покупатель / Получатель
- Выдан / Действует до
- Status badge (Активен/Истёк/Использован)
- Кнопка `Выдать сертификат` → toast «Сертификат создан · ссылка отправлена покупателю»

**Таб «Абонементы»** — 12 карточек:
- Аватар клиента + название пакета (на 4/8/12 визитов)
- Progress bar `usedVisits / totalVisits`
- Осталось дней (amber если <30)
- Оплаченная сумма

**Моки:** `loyalty.json`, `certificates.json`, `subscriptions.json`.

**4 программы из моков:**
1. **Кэшбэк AURA** — 7% начисление, до 30% списание (482 участника, 1.28M выдано, 624к использовано)
2. **Депозит +15%** — пополнение от 10к получает +15% (47 участников, 890к выдано)
3. **Сертификаты** — 142 выдано, 89 использовано, 612к оборот
4. **Абонементы** на 4/8/12 визитов со скидкой 15-25%

**Связи:**
- `bonusBalance` и `depositBalance` хранятся в `Client.bonusBalance / depositBalance`
- Сертификаты привязаны к buyerName (клиенту)

## 9. Финансы (`/finance`)

**Назначение:** P&L, расходы, ведомость зарплаты, прогноз.

**UI:**
- 4 stat-карточки: Выручка / Расходы / **Чистая прибыль** (emerald) / **Прогноз на след. месяц** (primary)
- 2 графика рядом:
  - **Денежный поток (ComposedChart)** — бары поступлений (зелёный) и расходов (красный) + линия баланса (primary)
  - **Структура расходов (PieChart)** — donut с легендой: ЗП / Расходники / Аренда / Налоги / Маркетинг / Прочее
- Табы: Зарплата мастеров / Прочие расходы

**Таб «Зарплата мастеров»** — таблица:
- Мастер (аватар + имя)
- Записей / Выручка мастера / Ставка % / Бонус / **К выплате** (жирно)
- Кнопка `Выплатить` → toast «412 000 ₽ переведено · Алина»
- Footer строка: Итого к выплате (жирно, крупно)

**Таб «Прочие расходы»** — таблица:
- Категория / Сумма / % от выручки

**Внизу** — primary блок «AI-прогноз и совет»:
- «Июнь принесёт ~1.34M. Чтобы держать маржу >28%, не превышайте ФОТ 564к»

**Моки:** `finance.json`, `payroll.json`.

**Расчёт в скрипте:**
```js
monthRevenue = sum(masters[].revenue) = 1.24M
monthCOGS = 14% от revenue
monthPayroll = sum(payroll.payout)
monthRent = 180k
monthMarketing = 35k
monthTaxes = 6% от revenue
monthProfit = monthRevenue - всё остальное
forecastNextMonth = monthRevenue × 1.08
```

**Связи:**
- Выплачивает Masters из `masters.json` через `payroll.json`
- Расходы складываются с `inventory.json` (расходники)

## 10. Склад (`/inventory`)

**Назначение:** учёт расходников, автосписание, заказы у поставщиков.

**UI:**
- Кнопки: `Заказать недостающее` / `Добавить позицию`
- 5 stat-карточек: Позиций / **Низкий остаток** (красная) / **Скоро истекут** (amber) / Стоимость склада / Расход за месяц
- Card с switch «Автосписание при оказании услуги» (primary блок)
- Search + табы: Все / Низкий остаток / Истекают
- Таблица 15 SKU:
  - Позиция (с Badge `Истекает` если expiringSoon) + дата истечения
  - Категория
  - **Остаток** — Progress bar (красный при low) + цифра + par-level
  - Стоимость / Расход за месяц / Поставщик
  - Кнопка `Заказать` → toast

**Низ страницы** — primary-блок AI-предложения:
- «Краска L'Oréal Majirel 6.0 закончится через 6 дней. Заказать 20 шт у BeautyProf — 19 600 ₽»
- Кнопки `Заказать 20 шт` / `Напомнить позже`

**Моки:** `inventory.json` (15 SKU: окрашивание, уход, ногти, косметология, брови, ресницы, расходники).

**Поля:**
- `inStock`, `parLevel`, `costPerUnit`, `monthConsumption`
- `lowStock = inStock < parLevel * 0.5` (флаг)
- `expiringSoon` если до `expiryDate` ≤30 дней

**Связи:**
- Косвенно связан с услугами (расходник списывается при услуге)
- Выручка с расходов в `finance.json` — `monthCOGS`

## 11. Аналитика (`/analytics`)

**Назначение:** глубокая аналитика — cohort retention, LTV, ABC, источники, health.

**UI:**
- 4 stat-карточки: Активная база / Retention M+3 / Средний LTV / **Health Score** (primary, 81/100)
- 5 табов:

**Таб «Cohort retention»** — heatmap-таблица:
- Строки: 8 когорт (окт 25 – май 26)
- Колонки: M+0 ... M+7
- Цвета ячеек по retention %:
  - ≥80% — emerald-500
  - ≥60% — emerald-400
  - ≥40% — amber-300
  - ≥20% — amber-400
  - >0% — red-300
- Внизу — AI-вывод: «35% теряются после первого визита, welcome-серия даёт +12 п.п. на M+1»

**Таб «LTV»** — BarChart распределения по бакетам:
- 0-10к / 10-30к / 30-60к / 60-100к / 100к+

**Таб «ABC»** — 3 stat-карточки (A/B/C класс) с количеством клиентов и долей выручки. Сортировка по LTV, кумулятивная доля → 80%/95%/100% границы.

**Таб «Источники»** — horizontal BarChart по каналам (телефон / Instagram / WhatsApp / рекомендации / 2ГИС / Яндекс.Карты), выручка с канала.

**Таб «Health»** — RadarChart по 6 метрикам:
- Удержание M+1 (65%)
- Возврат VIP (92%)
- Конверсия записи (78%)
- Завершение визита (96%)
- Загрузка слотов (76%)
- Активная база (81%)

**Низ страницы** — primary блок AI-инсайт о когорте фев 26 с конкретной суммой потерь.

**Моки:** `cohorts.json`, `dashboard.json`, `clients.json`.

**Связи:**
- ABC рассчитывается из `clients.ltv`
- Источники = `dashboard.sources`

## 12. Настройки (`/settings`)

## 13. Клиентский чат (`/client`)

**Назначение:** показать, как клиент взаимодействует с AI-ботом салона.

**UI:** мобильный mockup в рамке iPhone (на десктопе) / fullwidth (на мобайле):
- Шапка чата: эмодзи салона + название + статус «онлайн · отвечает мгновенно»
- 4 chip-таба сценариев: Записаться / Перенести / Напоминание / Вопросы
- Область сообщений (фон цвета салона):
  - Бот — серая бабблка слева
  - Пользователь — primary бабблка справа
  - Confirm-карточка (зелёная с ✓) в конце сценария
- Typing-индикатор (3 точки с framer-motion анимацией)
- Quick replies — кнопки внизу с border primary

**Стейт-машина сценария:**
```
idle → initial (первая реплика бота)
       → awaiting_user (показаны quick replies)
       → user clicks reply
       → bot_typing (1.2 сек typing-индикатор)
       → bot_response (новая реплика)
       → awaiting_user (новые quick replies или confirm)
       → completed (показана confirm-карточка)
```

**Сценарии:**
1. **Записаться** — выбор услуги (стрижка/окраш/маникюр/косметология) → выбор мастера → выбор времени → confirm-карточка
2. **Перенести** — текущая запись найдена → выбор нового времени → confirm
3. **Напоминание** — реактивация со скидкой → быстрая запись
4. **Вопросы** — FAQ: цены / часы / адрес / свободно завтра

**Переключение сценария** во время диалога → Dialog «Начать заново? Прогресс будет потерян»

**Моки:** `client_scenarios.json` (4 сценария с ветвлением).

**Связи:**
- Эмулирует то, что AI делает в `/inbox`
- Используется для **демонстрации клиенту**, как он будет общаться с ботом

## Маршрут пользователя на демо (как смотреть)

Стандартный демо-сценарий для салона:
1. Открываем `/` — за 5 сек видим экономику (KPI + Катя в минусе + 3 инсайта)
2. `/inbox` — «AI отвечает за вашего админа»
3. `/reactivation` — «вот 47 ваших клиентов которых вы теряете»
4. Переключаем в `Клиент` → видим UX со стороны клиента
5. Возвращаемся → `/schedule` — журнал
6. `/marketing` — A/B-тесты, выручка от триггеров
7. `/loyalty` — программы удержания
8. `/finance` — P&L и прогноз
9. `/analytics` — cohort retention и health
10. `/settings` — «давайте настроим под ваш салон»

## Расширение функционала

Хочешь добавить ещё разделы (например, **Расписание мастеров с отпусками** или **Электронная медкарта** для стоматологии)?

1. Создай тип в `src/types/index.ts`
2. Сгенерируй моки в `scripts/gen-mocks-extra.mjs` и запусти его
3. Добавь функцию `getX()` в `src/lib/data/index.ts`
4. Создай страницу `src/pages/X.tsx` (используй любую из существующих как шаблон)
5. Добавь маршрут в `src/App.tsx`
6. Добавь пункт в `Sidebar.tsx`

Шаблоны компонентов по сложности:
- Простая страница со списком — `Inventory.tsx` (table + filters + AI блок)
- Страница с табами — `Settings.tsx` (6 табов)
- Страница с графиками — `Finance.tsx` (composed + pie + table)
- Страница с детальным sheet — `Clients.tsx` или `Schedule.tsx`
